# 画面設計書

## 1. 画面一覧

| 画面ID | 画面名 | URL | 説明 |
|--------|--------|-----|------|
| SCR-01 | カンバンボード | `/` | アプリケーションのメイン画面。全機能をこの1画面で提供する |

本アプリケーションは SPA（Single Page Application）であり、ルーティングは存在しない。すべての操作はカンバンボード画面上で完結する。

### 2.2 コンポーネント構成

```
App
├── ヘッダー（静的要素）
└── メインコンテンツ
    ├── FilterBar           # ステータスフィルター
    ├── [+ 新規タスク] ボタン
    ├── TaskForm            # フォーム表示時のみレンダリング
    ├── ローディング表示     # API取得中のみ
    ├── エラー表示          # API失敗時のみ
    └── Board               # ドラッグ&ドロップコンテキスト

        │   └── TaskCard × N

        │   └── TaskCard × N
        └── Column（完了）
            └── TaskCard × N
```

モーダルは `TaskCard` の子要素としてレンダリングされる（DOM 上は `App` と並列）。

```
TaskCard
└── TaskDetailModal  # カードクリック時のみレンダリング
```

### 3.2 FilterBar（フィルターバー）

**ファイル:** [FilterBar.tsx](../frontend/src/components/FilterBar.tsx)

```
[ すべて ]  [ Todo ]  [ 進行中 ]  [ 完了 ]   [+ 新規タスク]
```

#### フィルターボタン仕様

| ラベル | `filter` 値 | API 呼び出し |
|--------|------------|------------|
| すべて | `all` | `GET /api/tasks` |
| Todo | `todo` | `GET /api/tasks/status/todo` |
| 進行中 | `in_progress` | `GET /api/tasks/status/in_progress` |
| 完了 | `done` | `GET /api/tasks/status/done` |

- 選択中のボタンは青背景・白文字（`bg-blue-600 text-white`）
- 未選択ボタンは白背景・グレー文字・グレーボーダー
- デフォルト選択は「すべて」

#### 新規タスクボタン

- フォーム非表示時: 「+ 新規タスク」（青背景）
- フォーム表示時: 「閉じる」（同じ見た目）
- クリックで `TaskForm` の表示/非表示を切り替える

```
┌──────────────────────────────────────────┐

│ ┌──────────────────────────────────────┐ │
│ │                                      │ │
│ └──────────────────────────────────────┘ │
│                                          │
│ 説明                                     │
│ ┌──────────────────────────────────────┐ │
│ │                                      │ │
│ │                                      │ │

│ └──────────────────────────────────────┘ │

└──────────────────────────────────────────┘
```

#### フォームフィールド定義

| フィールド | 入力形式 | 必須 | 制約 | `id` 属性 |
|-----------|---------|------|------|----------|
| タイトル | text input | 必須（`*` 表示） | `maxLength=255` | `new-title` |
| 説明 | textarea（3行） | 任意 | — | `new-description` |
| 優先度 | select | 任意 | 未設定 / 高 / 中 / 低 | `new-priority` |
| 期日 | date input | 任意 | `YYYY-MM-DD` | `new-due-date` |

#### ボタン

| ボタン | 動作 |
|--------|------|
| キャンセル | フォームを閉じる（`TaskForm` をアンマウント） |
| 作成 | `POST /api/tasks` を呼び出す。成功後フォームを閉じてタスク一覧を再取得 |

#### バリデーション・エラー表示

- タイトルが空白のままで「作成」を押すとフォーム内にエラーメッセージを表示
- API エラー時もフォーム内にエラーメッセージを表示
- 送信中はボタンを「保存中...」に変更し、`disabled` にする

### 3.4 Board（カンバンボード）

**ファイル:** [Board.tsx](../frontend/src/components/Board.tsx)

3カラムを横並びに表示し、ドラッグ&ドロップのコンテキストを提供する。

#### カラム定義

| カラム | `status` 値 | ヘッダーカラー |
|--------|------------|-------------|
| Todo | `todo` | グレー（`bg-gray-400`） |
| 進行中 | `in_progress` | 青（`bg-blue-400`） |
| 完了 | `done` | 緑（`bg-green-400`） |

#### ドラッグ&ドロップ動作

1. カードをドラッグ開始 → カードを回転・半透明表示
2. ドロップ先カラムにホバー → カラム背景を青くハイライト
3. ドロップ → 楽観的 UI 更新（即時反映）
4. API 呼び出し（非同期）
   - カラム間移動: `PATCH /api/tasks/{id}/status` → `PATCH /api/tasks/{id}/position`
   - 同カラム内: `PATCH /api/tasks/{id}/position` のみ
5. API 失敗時 → ドラッグ前の状態にロールバック

### 3.6 TaskCard（タスクカード）

**ファイル:** [TaskCard.tsx](../frontend/src/components/TaskCard.tsx)

```
通常状態:
┌────────────────────────────────┐
│ タスクタイトル                  │
│ 説明テキスト（1行 truncate）    │
│ [高] 2026-05-01               │
└────────────────────────────────┘

ホバー状態:
┌────────────────────────────────┐
│ タスクタイトル          [🗑]   │  ← ゴミ箱ボタン出現
│ 説明テキスト（1行 truncate）    │
│ [高] 2026-05-01               │
└────────────────────────────────┘

ドラッグ中:
┌────────────────────────────────┐  ← 影・回転・半透明
│ タスクタイトル                  │
│ 説明テキスト（1行 truncate）    │
│ [高] 2026-05-01               │
└────────────────────────────────┘
```

#### 状態別スタイル

| 状態 | 適用スタイル |
|------|------------|
| 通常 | `shadow-sm`、白背景、グレーボーダー |
| ホバー | `shadow-md`、ゴミ箱ボタン表示 |
| ドラッグ中 | `shadow-lg rotate-1 opacity-90` |

#### 表示要素

| 要素 | 表示条件 | 備考 |
|------|---------|------|
| タイトル | 常時 | — |
| 説明 | `description` が非 null かつ非空 | 1行 truncate |
| 優先度バッジ | `priority` が非 null | 色は[優先度値](#優先度バッジ配色)参照 |
| 期日 | `dueDate` が非 null | 期日超過時は赤色・太字 |
| ゴミ箱ボタン | ホバー時のみ（`opacity-0` → `opacity-100`） | クリックで確認ダイアログ後に削除 |

#### 優先度バッジ配色

| priority | 背景色 | 文字色 | 表示ラベル |
|----------|--------|--------|-----------|
| `high` | `bg-red-100` | `text-red-700` | 高 |
| `medium` | `bg-yellow-100` | `text-yellow-700` | 中 |
| `low` | `bg-green-100` | `text-green-700` | 低 |

#### 期日超過判定ロジック

```
期日超過 = (dueDate !== null) AND (dueDate < 今日の日付) AND (status !== 'done')
```

#### クリック動作

- カード本体クリック → `TaskDetailModal` を開く
- ゴミ箱ボタンクリック → 確認ダイアログ（`window.confirm`）→ 確認後 `DELETE /api/tasks/{id}` 呼び出し

## 4. 状態管理

状態は `App` コンポーネントで一元管理する。子コンポーネントにはコールバック関数でデータ変更を通知する。

```
App
├── tasks: TaskResponse[]         # タスク一覧
├── selectedStatus: TaskFilter    # 選択中フィルター ('all' | 'todo' | 'in_progress' | 'done')
├── loading: boolean              # API ローディング状態
├── error: string | null          # API エラーメッセージ
└── showForm: boolean             # タスク登録フォーム表示状態
```

### コールバック関数

| 関数名 | 型 | 呼び出し元 | 動作 |
|--------|-----|---------|------|
| `handleUpdated` | `(task: TaskResponse) => void` | Board → TaskCard → TaskDetailModal | 特定タスクを新しいオブジェクトで置き換える |
| `handleReordered` | `(tasks: TaskResponse[]) => void` | Board | タスク配列全体を置き換える（DnD 後） |
| `handleDeleted` | `(id: number) => void` | Board → TaskCard | 指定 ID のタスクを除去する |
| `handleCreated` | `() => void` | TaskForm | フォームを閉じてタスク一覧を再取得する |

---

## 5. ローディング・エラー表示

| 状態 | 表示内容 | 表示位置 |
|------|---------|---------|
| タスク取得中 | 「読み込み中...」（グレーテキスト） | ボードの上 |
| タスク取得失敗 | 「タスクの取得に失敗しました」（赤テキスト） | ボードの上 |
| ローディング中・エラー時 | Board は非表示 | — |

## 6. レスポンシブ対応

現在の実装はレスポンシブ対応を行っていない。3カラムは常に横並び（`flex`）で表示され、画面幅が狭い場合は横スクロールが発生する。

## 7. 画面フロー

```
アプリ起動
    │
    ▼
タスク一覧取得（GET /api/tasks）
    │
    ▼
カンバンボード表示
    │
    ├── フィルター変更 ──────────→ タスク再取得 → ボード更新
    │
    ├── 「+ 新規タスク」クリック
    │       │
    │       ▼
    │   タスク登録フォーム表示
    │       │
    │       ├── 「キャンセル」→ フォームを閉じる
    │       │
    │       └── 「作成」→ POST /api/tasks → 成功: タスク再取得 → フォームを閉じる
    │
    ├── タスクカードクリック
    │       │
    │       ▼
    │   タスク編集モーダル表示
    │       │
    │       ├── 「×」「キャンセル」「背景クリック」→ モーダルを閉じる
    │       │
    │       └── 「保存」→ PUT /api/tasks/{id} → 成功: カード即時更新 → モーダルを閉じる
    │
    ├── タスクカードをドラッグ&ドロップ
    │       │
    │       ├── 同カラム内 ────→ 楽観的更新 → PATCH /api/tasks/{id}/position
    │       │
    │       └── カラム間 ──────→ 楽観的更新 → PATCH /api/tasks/{id}/status
    │                                        → PATCH /api/tasks/{id}/position
    │
    └── ゴミ箱ボタンクリック
            │
            ▼
        確認ダイアログ
            │
            ├── キャンセル → 何もしない
            │
            └── OK → DELETE /api/tasks/{id} → 成功: カードを一覧から除去
```
