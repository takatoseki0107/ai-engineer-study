# 要件定義書

## 1. プロジェクト概要

タスクをカンバンボード形式で管理する Web アプリケーション。個人またはチームがタスクの進捗を「Todo / 進行中 / 完了」の3ステータスで可視化し、ドラッグ&ドロップで直感的に操作できる。

---

## 2. 機能要件

### 2.1 タスク管理

#### FR-01 タスク一覧表示

- カンバンボード形式（3カラム）でタスクを一覧表示する
- カラムはステータスごとに分かれており、左から「Todo」「進行中」「完了」の順で並ぶ
- 各カラム内のタスクは `position` 昇順で表示する
- 各カラムのヘッダーにタスク件数バッジを表示する
- タスクが0件のカラムには「タスクなし」を表示する

#### FR-02 タスク新規登録

- 画面上部の「+ 新規タスク」ボタンを押すと登録フォームをインライン表示する
- 入力項目は以下の通り

  | 項目 | 必須 | 制約 |
  |------|------|------|
  | タイトル | 必須 | 255文字以内 |
  | 説明 | 任意 | 制限なし |
  | 優先度 | 任意 | high / medium / low のいずれか |
  | 期日 | 任意 | YYYY-MM-DD 形式 |

- タイトル未入力のまま送信した場合はバリデーションエラーを表示する
- 登録成功後、フォームを閉じてタスク一覧を再取得する

#### FR-03 タスク編集

- タスクカードをクリックするとモーダルダイアログで編集フォームを開く
- 編集可能な項目はタイトル・説明・優先度・期日の4項目
- バリデーションルールは FR-02 と同様
- 保存成功後、モーダルを閉じてカード表示を即時更新する（再取得なし）
- モーダル背景クリックまたは × ボタンで編集をキャンセルできる

#### FR-04 タスク削除

- タスクカードにホバーするとカード右上にゴミ箱ボタンが表示される
- ゴミ箱ボタンを押すと確認ダイアログ（`window.confirm`）を表示する
- 確認後、タスクを物理削除し、一覧から即時除去する
- 削除 API 失敗時はアラートでエラーを通知する

#### FR-05 ドラッグ&ドロップによるステータス変更・並び替え

- タスクカードをドラッグして別カラムに移動するとステータスが変更される
- 同じカラム内でのドラッグで表示順を変更できる
- ドラッグ中はカードを回転・半透明表示にして視覚的フィードバックを与える
- ドロップ先カラムは背景色でハイライトする
- UI は楽観的更新（ドロップ直後に画面を反映）し、API 呼び出しを非同期で行う
- API 失敗時はドラッグ前の状態にロールバックする

#### FR-06 ステータスフィルター

- 画面上部のフィルターバーで「すべて / Todo / 進行中 / 完了」を切り替えられる
- フィルター選択時は選択ステータスのタスクのみボード上に表示する
- デフォルトは「すべて」

#### FR-07 期日超過ハイライト

- 期日が設定されており、かつ今日の日付を過ぎており、かつステータスが「完了」以外のタスクは期日を赤色・太字で表示する

---

### 2.2 認証

#### FR-08 ユーザー登録

- ユーザー名とパスワードを入力して新規アカウントを作成できる
- ユーザー名は一意であり、重複登録はエラーになる
- パスワードは BCrypt でハッシュ化して保存する
- 登録成功後は JWT トークンを発行してログイン状態にする

#### FR-09 ログイン

- 登録済みのユーザー名・パスワードで認証できる
- 認証成功時は JWT トークンを返し、フロントエンドは `localStorage` に保存する
- 以降のリクエストには `Authorization: Bearer <token>` ヘッダーを付与する

#### FR-10 ログアウト

- ヘッダーのログアウトボタンを押すと `localStorage` のトークンを削除してログイン画面へ遷移する
- ログアウト後はタスク管理画面にアクセスできない（`ProtectedRoute` でリダイレクト）

---

### 2.3 タスクカード表示仕様

各タスクカードには以下の情報を表示する。

| 要素 | 条件 |
|------|------|
| タイトル | 常時表示 |
| 説明（truncate） | 説明が設定されている場合のみ |
| 優先度バッジ | 優先度が設定されている場合のみ |
| 期日 | 期日が設定されている場合のみ。期日超過時は赤色表示 |
| ゴミ箱ボタン | カードホバー時のみ表示 |

優先度バッジの配色：

| 優先度 | 表示ラベル | 配色 |
|--------|-----------|------|
| high | 高 | 赤背景・赤文字 |
| medium | 中 | 黄背景・黄文字 |
| low | 低 | 緑背景・緑文字 |

### 3.2 アーキテクチャ

```
ブラウザ (React + Vite :5173)
    │  /api/* → Vite Proxy
    ▼
バックエンド (Spring Boot :8080)
    │  JPA
    ▼
PostgreSQL (:5432)
```

- フロントエンドの `/api/*` リクエストは Vite 開発サーバーが `http://localhost:8080` へプロキシする
- CORS 設定によりフロントエンドオリジン（`http://localhost:5173`）からのアクセスを許可する
- フロントエンドとバックエンドは完全に分離した SPA 構成

---

### 3.3 データ永続化

- タスクデータ・ユーザーデータは PostgreSQL に永続化する
- アプリケーション起動時に `db/init.sql` でスキーマを投入する（`IF NOT EXISTS` によるべき等性確保）
- `db/data.sql` は認証導入によりサンプルデータを削除済みで、現在は無効化されている
- JPA による ORM でデータアクセスを行う

---

### 3.4 バリデーション

| レイヤー | 対象 | 内容 |
|---------|------|------|
| フロントエンド | タイトル | 必須・255文字以内（`maxLength` 属性） |
| バックエンド | タイトル | `@NotBlank`・`@Size(max=255)` |
| バックエンド | ステータス | `@NotBlank` |
| バックエンド | ポジション | `@Min(1)` |
| バックエンド | ユーザー名・パスワード | `@NotBlank` |
| DB | priority | CHECK 制約（`high` / `medium` / `low`） |
| DB | status | CHECK 制約（`todo` / `in_progress` / `done`） |

---

### 3.5 エラーハンドリング

| エラー種別 | HTTP ステータス | レスポンス形式 |
|-----------|---------------|---------------|
| タスクが存在しない | 404 Not Found | `{ "error": "メッセージ" }` |
| バリデーション違反 | 400 Bad Request | `{ "error": "メッセージ" }` |
| 不正な優先度・ステータス値 | 400 Bad Request | `{ "error": "メッセージ" }` |
| ユーザー名重複 | 400 Bad Request | `{ "error": "メッセージ" }` |
| 認証失敗（ユーザー名・パスワード不一致） | 401 Unauthorized | — |
| JWT トークン未提供・無効 | 401 Unauthorized | — |

フロントエンドは API エラー発生時に UI 上にエラーメッセージを表示する。  
削除失敗時のみ `alert()` でユーザーに通知する。

- フォーム要素は `<label htmlFor>` と `<input id>` を対応付け、アクセシビリティを確保する

## 4. データモデル

### users テーブル

| カラム | 型 | 制約 | 説明 |
|--------|-----|------|------|
| id | BIGSERIAL | PK, AUTO INCREMENT | ユーザー識別子 |
| username | VARCHAR(50) | NOT NULL, UNIQUE | ユーザー名 |
| password | VARCHAR(255) | NOT NULL | BCrypt ハッシュ済みパスワード |
| created_at | TIMESTAMP | NOT NULL | 作成日時 |

### tasks テーブル

| カラム | 型 | 制約 | 説明 |
|--------|-----|------|------|
| id | BIGINT | PK, AUTO INCREMENT | タスク識別子 |
| user_id | BIGINT | NOT NULL, FK → users(id) | 所有ユーザー |
| title | VARCHAR(255) | NOT NULL | タスクタイトル |
| description | TEXT | NULL 可 | 詳細説明 |
| priority | VARCHAR(10) | NULL 可, CHECK | `high` / `medium` / `low` |
| status | VARCHAR(20) | NOT NULL, CHECK | `todo` / `in_progress` / `done` |
| due_date | DATE | NULL 可 | 期日 |
| position | INTEGER | NOT NULL | カラム内の表示順（1始まり） |
| created_at | TIMESTAMP | NOT NULL | 作成日時（更新不可） |
| updated_at | TIMESTAMP | NOT NULL | 最終更新日時 |

インデックス：

| インデックス | カラム | 用途 |
|------------|--------|------|
| idx_tasks_user_id | user_id | ユーザー絞り込み |
| idx_tasks_status | status | ステータス絞り込み |
| idx_tasks_status_position | (status, position) | カラム内ソート |
| idx_tasks_due_date | due_date | 期日検索 |
